## Project title

Blockscout explorer

## Short description

Blockscout is an open-source blockchain explorer tailored for Ethereum and EVM-compatible networks, leveraging Elixir and Phoenix Framework for high concurrency, real-time updates, and low-latency performance. Its modular architecture utilizes a robust ETL pipeline for blockchain data ingestion and indexing, with support for trace-based token detection, internal transaction tracing, and multi-chain compatibility. The platform features comprehensive smart contract verification, including Sourcify integration and support for complex formats like standard-json-input and multi-part files. Developers benefit from REST/GraphQL APIs, customizable parameters for non-standard chains, and advanced caching strategies (Redis) for high-throughput environments. Blockscout supports multiple token standards (ERC-20, ERC-721, ERC-1155), customizable UIs, and deployment flexibility across public and private blockchains, making it a scalable and versatile tool for blockchain exploration.

## Additional information

The workspace contains only the backend server of the Blockscout explorer project. The backend server consists of several major components:

- **Indexer** (files: `apps/indexer/**/*`, top module: `Indexer`) is an Elixir-based backend process that uses supervised `GenServers` to fetch blockchain data through an advanced ETL pipeline. It efficiently handles both real-time and catch-up indexing by buffering and batching data, ensuring optimal concurrency and memory usage. After transforming raw blockchain data into structured formats, it forwards the prepared data to the Explorer module for storage and further processing. Its design includes specialized modules for multi-chain support, robust error handling, and data retry mechanisms.

- **API server** (files: `apps/block_scout_web/**/*`, top module: `BlockScoutWeb`) is built on the Phoenix framework and serves as the primary interface for accessing blockchain data. It exposes both RESTful and GraphQL APIs that allow clients to retrieve resources such as addresses, transactions, blocks, and tokens. Real-time updates are delivered using Phoenix Channels, and custom plugs manage rate limiting, API versioning, and logging.

- **Explorer** (files: `apps/explorer/**/*`, top module: `Explorer`) stores, processes, and serves blockchain data ingested by the Indexer. It provides a high-level data access layer through modules like `Explorer.Chain` and `Explorer.Repo` that interface with a PostgreSQL database, with schema migrations managed in `apps/explorer/priv/repo/migrations`. It features a structured multi-stage ETL pipeline used by the Indexer to import and transform blockchain data. The system includes chain-specific modules and configurations tailored for various EVM networks. It supports smart contract verification by managing compilation, versioning, and detecting proxy patterns. On-demand data fetching is available and is invoked by the `BlockScoutWeb` API server to support features such as contract verification. Release tasks are incorporated for database setup and migrations. Additional modules handle account management, market data processing, and event subscriptions to keep clients updated on blockchain data changes. Specialized modules manage both schema and long-running data migrations, including backfilling and index optimizations.

- **Ethereum JSON RPC** (files: `apps/ethereum_jsonrpc/**/*`, top module: `EthereumJSONRPC`) is an Elixir library that interacts with Ethereum and EVM-compatible blockchains via JSON-RPC APIs. It constructs and sends both single and batch JSON-RPC requests while managing retries, timeouts, and throttling. The component abstracts transport mechanisms using a callback-based behaviour that supports HTTP, IPC, and WebSockets. It adapts to various Ethereum clients by implementing a variant behaviour for each, such as Geth, Nethermind, Erigon, Besu, and others. The library encodes function calls and decodes responses in accordance with ABI specifications. It retrieves diverse blockchain data including blocks, transactions, receipts, logs, and account balances. Additionally, it supports real-time updates through WebSocket subscriptions and monitors endpoint availability to switch to fallback URLs when needed. Request coordination is enhanced by a rolling window mechanism that prevents overwhelming the JSON-RPC node.
